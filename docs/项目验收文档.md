### 7.4 数据完整性

每个监视项应具备：
- [x] 1 个 `source_no` 指向主数据源
- [x] 2 个数据源记录 (`ssmcsourcemst`)
- [x] 1 个区间时长规则 (`ssmcrulerandurmst`)
- [x] RuleType = 2 (RangeDuration)
- [x] MonitorStatus = `39edc1419d86d38933a57f1e156b4991` (考核条件主题)

### 7.5 关键配置要求 (重要!)

**org_no 一致性：** `ssmcitemmst.org_no`、`ssmcsourcemst.org_no`、`ssmcrulerandurmst.org_no` 必须一致。MonitorCenter 的 ABP 多租户过滤器 (`IMayHaveTenant`) 按 `org_no` 过滤，不一致会导致 `GetAllMonitors()` 查不到数据源和规则。

**表关联关系：**
```
ssmcitemmst.monitor_no ← ssmcsourcemst.group_no  (数据源关联)
ssmcitemmst.monitor_no ← ssmcrulerandurmst.related_no  (规则关联)
ssmcitemmst.source_no  → ssmcsourcemst.source_no  (关注数据源)
```

**数据源要求 (每个监视项 2 个)：**
| source_flag | source_id 格式 | source_cod | 说明 |
|-------------|---------------|------------|------|
| 3 (Trend) | `tag_{monitor_id}` | `db05.t{N}` | 趋势测点 |
| 1 (Static) | `threshold_{monitor_id}` | `80` | 阈值常量 |

**规则要求 (每个监视项 1 个 RangeDuration)：**
| 字段 | 值 |
|------|-----|
| `ridur_no` | `ridur_{monitor_no}` |
| `related_no` | `{monitor_no}` |
| `left_id` | `tag_{monitor_id}` (匹配趋势数据源 source_id) |
| `right_id` | `threshold_{monitor_id}` (匹配阈值数据源 source_id) |
| `symbol_flag` | 1 (大于) |
| `duration_cnt` | 30 (持续 30 秒) |
| `statuslin_cod` | `satisfiled` |
| `status_no` | `39edc1419d86d38933a57f1e156b4991` |

**TagName 解析 (GetAllMonitors.go)：** `TagName` 不能直接用 `source_no`，需解析为关注数据源的 `source_id`：
```csharp
var focusSrc = srcs?.FirstOrDefault(s => s.Id == m.FocusSourceId);
var resolvedTagName = focusSrc?.Alias ?? m.FocusSourceId ?? "";
```

**重建数据源/规则的 SQL 模板：** 见 `docs/test-data-template.sql`

### 7.6 数据验证命令

```bash
# 检查每个监视项的数据源和规则是否完整
docker exec sis-postgres psql -U postgres -d sis1 -c "
SELECT i.monitor_no,
       (SELECT COUNT(*) FROM ssmcsourcemst s WHERE s.group_no = i.monitor_no) as src_count,
       (SELECT COUNT(*) FROM ssmcrulerandurmst r WHERE r.related_no = i.monitor_no) as rule_count
FROM ssmcitemmst i
WHERE i.enable_flag = true
AND (
    (SELECT COUNT(*) FROM ssmcsourcemst s WHERE s.group_no = i.monitor_no) != 2
    OR (SELECT COUNT(*) FROM ssmcrulerandurmst r WHERE r.related_no = i.monitor_no) != 1
);"

# 检查 org_no 一致性
docker exec sis-postgres psql -U postgres -d sis1 -c "
SELECT 'ssmcitemmst' AS tbl, org_no, COUNT(*) FROM ssmcitemmst WHERE enable_flag = true GROUP BY org_no
UNION ALL
SELECT 'ssmcsourcemst', org_no, COUNT(*) FROM ssmcsourcemst GROUP BY org_no
UNION ALL
SELECT 'ssmcrulerandurmst', org_no, COUNT(*) FROM ssmcrulerandurmst GROUP BY org_no;"
```

## 九、已修复缺陷清单

| # | 缺陷 | 修复方式 |
|---|------|----------|
| D1 | 重复 clear 事件 | clear 后重置 `state.PreviousStatus = null`, `state.PreviousEventOccurTime = null` |
| D2 | clear_time UPDATE 无精确行匹配 | 使用 `RelatedTriggerOccurTime` 进行 `occur_time =` 精确匹配 |
| A3 | RelatedStatusName 显示事件状态名 | 分布式模式从 MonitorStatus 表查询 Name |
| A4 | OccurMonitorStatusName 为空 | 分布式模式从 MonitorStatusDefinition 表查询 Name |
| A5 | totalCount 与实际不一致 | 传递 `monitorKeys` 到 Rule Engine 查询 |
| B1 | 持续时长可能为负数 | Worker 侧精确匹配 + MonitorCenter 侧负值守卫 |
| B3 | totalCount 与实际不一致 | 传递 `monitorKeys` 到 Rule Engine 查询 |
| C1 | 本地模式分页未生效 | `GetAllForList` 和 `GetAllKeysForList` 添加 Skip/Take |
| A1/A2 | lastEventName/unit 等字段为空 | ClickHouse schema + Model + DTO + Writer + Service 全链路添加新字段 |
| D3 | MonitorCenter GetAllMonitors 返回空数据源/规则 | `org_no` 不一致导致 ABP tenant filter 过滤掉 sources/rules — 统一设为 `-1` |
| D4 | 监视项 9-100 无数据源和规则 | 重建 sources(2/监视项) + rules(1/监视项)，修正 `related_no`/`group_no` 匹配 |
| D5 | TagName 被设为 source_no 而非趋势测点名 | GetAllMonitors 解析关注数据源的 Alias 作为 TagName |
| D6 | 产生大量虚假 clear 事件、0 trigger | `CalcRuleRangeDuration` 和 `ProcessMonitorAsync` 共用 state key 导致碰撞 — 用 `{id}:rdur` 隔离 |
