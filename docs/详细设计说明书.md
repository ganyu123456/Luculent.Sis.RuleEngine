# 规则引擎（RuleEngine）详细设计说明书

## 1. 模块初始化流程

### 1.1 Master 启动时序

```
RuleEngine.Master 启动
    │
    ├── Program.cs: Host.CreateDefaultBuilder()
    │   ├── 注册 gRPC 服务 (WorkerCommunicationService)
    │   ├── 注册 HTTP 端点 (Minimal API)
    │   └── 注册后台服务:
    │       ├── ConfigurationSyncService    (配置同步)
    │       ├── WorkerHealthCheckService    (Worker 心跳检测)
    │       └── LeaderElectionService      (Redis 选主)
    │
    └── LeaderElectionService 获取 Leader 锁后:
        ├── 从 MonitorCenter 拉取全量配置
        ├── ConfigurationService.LoadFullConfig(monitors)
        ├── PartitionService.Partition(monitors, workerCount)
        ├── WorkerManager.AssignMonitors() → gRPC 推送给各 Worker
        └── LeaderElectionService 开始续约循环 (每 5s)
```

### 1.2 Worker 启动时序

```
RuleEngine.Worker 启动 (Native AOT binary)
    │
    ├── Program.cs: Host.CreateDefaultBuilder()
    │   ├── 注册后台服务:
    │   │   └── WorkerCalculationService   (主计算循环)
    │   └── 注册单例服务:
    │       ├── RocksDbStateStore          (本地状态持久化)
    │       ├── RedisAlarmWriter            (Redis 写入)
    │       ├── ClickHouseAlarmWriter       (ClickHouse 写入)
    │       ├── TrendDbReader               (TrendDB 连接池)
    │       └── MasterConnectionService    (gRPC 客户端)
    │
    └── MasterConnectionService 启动:
        ├── 连接 Master gRPC 服务
        ├── 发送 Register 请求 (workerId, capacity)
        ├── 接收 AssignMonitors 响应 (初始监控项 + 规则配置)
        ├── 加载规则到本地 ConcurrentDictionary
        ├── 从 RocksDB 恢复上一次计算状态
        └── 启动主计算循环
```

```csharp
/// <summary>
/// 纯计算基类，不依赖 MonitorCenter 的任何基类。
/// </summary>
public abstract class RuleCalculatorBase
{
    protected readonly ILogger Logger;

    protected RuleCalculatorBase(ILogger logger)
    {
        Logger = logger;
    }

    /// <summary>
    /// 比较左右值。纯函数，直接翻译自 MonitorCenter 的 CompareSymbol。
    /// </summary>
    protected static bool CompareSymbol(double left, double right, SymbolType symbol)
    {
        return symbol switch
        {
            SymbolType.Greater => left > right,
            SymbolType.GreaterOrEqual => left >= right,
            SymbolType.Less => left < right,
            SymbolType.LessOrEqual => left <= right,
            _ => false,
        };
    }

    /// <summary>
    /// 从数据字典中直接获取测点值（无别名解析，TrendDB 返回的就是最终值）。
    /// </summary>
    protected static double? GetValue(IDictionary<string, double?> data, string tagName)
    {
        return data.TryGetValue(tagName, out var val) ? val : null;
    }
}
```

#### 2.3.2 区间时长计算器

```csharp
/// <summary>
/// 区间时长规则计算器。从 MonitorCenter CalculateRuleRangeDuration 提取纯计算逻辑。
/// 状态存储改为通过 IStateStore 接口（RocksDB 实现），不再使用静态 ConcurrentDictionary。
/// </summary>
public class CalculateRuleRangeDuration : RuleCalculatorBase
{
    private readonly IStateStore _stateStore;

    public CalculateRuleRangeDuration(ILogger<CalculateRuleRangeDuration> logger, IStateStore stateStore)
        : base(logger)
    {
        _stateStore = stateStore;
    }

    public async Task<RuleCalculateResult> CalculateAsync(
        MonitorConfig monitor,
        List<RangeDurationRuleConfig> rules,
        IDictionary<string, double?> data,
        DateTime? calcTime = null)
    {
        var result = new RuleCalculateResult();
        var time = calcTime ?? DateTime.UtcNow;
        var timeMs = new DateTimeOffset(time).ToUnixTimeMilliseconds();

        if (rules.Count == 0)
        {
            result.IsSuccess = true;
            result.Logs.Add("没有已启用的区间时长规则");
            return result;
        }

        var state = await _stateStore.GetAsync(monitor.Id);

        foreach (var rule in rules.Where(r => r.IsEnabled))
        {
            var left = GetValue(data, rule.LeftTagName);
            var right = GetValue(data, rule.RightTagName);

            if (left == null || right == null) continue;

            var isSatisfied = CompareSymbol(left.Value, right.Value, rule.SymbolType);

            // 无需时长判断，直接返回
            if (rule.DurationSecond <= 0)
            {
                if (isSatisfied)
                {
                    result.State = rule.StatusKey;
                    result.HasEvent = true;
                    if (rule.BreakOnHit) break;
                }
                continue;
            }

            // 需要时长判断
            var durationHit = await CheckDurationHit(
                monitor.Id, rule.StatusKey, isSatisfied, timeMs, rule.DurationSecond, state);

            if (durationHit)
            {
                result.State = rule.StatusKey;
                result.HasEvent = true;
                if (rule.BreakOnHit) break;
            }
        }

        await _stateStore.SaveAsync(monitor.Id, state);
        return result;
    }

    private async Task<bool> CheckDurationHit(
        string monitorId, string statusKey, bool hit,
        long timeMs, long durSecond, CalculationState state)
    {
        if (string.IsNullOrEmpty(statusKey))
            throw new ArgumentException("statusKey 不能为空");

        // 首次计算
        if (state == null || state.PreviousStatus == null)
        {
            state = new CalculationState
            {
                MonitorId = monitorId,
                RuleType = RuleType.RangeDuration,
                LastSatisfiedTimeMs = hit ? timeMs : 0,
                AccumulatedDurationSec = 0,
                PreviousStatus = hit ? statusKey : string.Empty,
                ConfigVersion = DateTime.UtcNow,
            };
            return false;
        }

        if (!hit)
        {
            state.LastSatisfiedTimeMs = 0;
            state.AccumulatedDurationSec = 0;
            return false;
        }

        // 已经在满足状态中
        if (state.PreviousStatus == statusKey && state.LastSatisfiedTimeMs > 0)
        {
            var elapsedSec = TimeSpan.FromMilliseconds(timeMs - state.LastSatisfiedTimeMs).TotalSeconds;
            if (elapsedSec >= durSecond)
            {
                return true;
            }
        }
        else
        {
            // 刚开始满足条件
            state.LastSatisfiedTimeMs = timeMs;
            state.AccumulatedDurationSec = 0;
            state.PreviousStatus = statusKey;
        }

        return false;
    }
}
```

#### 2.3.3 Worker 主计算循环

```csharp
/// <summary>
/// Worker 的主计算循环。注册为 BackgroundService，由 .NET Host 管理生命周期。
/// </summary>
public class WorkerCalculationService : BackgroundService
{
    private readonly ITrendDataReader _trendReader;
    private readonly IStateStore _stateStore;
    private readonly IAlarmWriter _alarmWriter;
    private readonly RuleDispatcher _dispatcher;
    private readonly ConcurrentDictionary<string, MonitorConfig> _assignedMonitors = new();
    private readonly IMasterConnection _masterConnection;
    private readonly ILogger<WorkerCalculationService> _logger;

    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        // 注册到 Master
        await _masterConnection.RegisterAsync(new WorkerInfo { ... });

        // 订阅 Master 推送
        _masterConnection.OnAssigned += monitors =>
        {
            foreach (var m in monitors)
                _assignedMonitors[m.Id] = m;
        };
        _masterConnection.OnRevoked += ids =>
        {
            foreach (var id in ids)
                _assignedMonitors.TryRemove(id, out _);
        };

        // 主计算循环
        while (!ct.IsCancellationRequested)
        {
            try
            {
                var tagNames = _assignedMonitors.Values.Select(m => m.TagName).Distinct().ToList();
                if (tagNames.Count == 0)
                {
                    await Task.Delay(1000, ct);
                    continue;
                }

                var values = await _trendReader.ReadBatchAsync(tagNames);
                var now = DateTime.UtcNow;

                // 按刷新间隔筛选到期监控项，按规则类型分组
                var dueMonitors = _assignedMonitors.Values
                    .Where(m => (now - m.LastCalculateTime).TotalSeconds >= m.RefreshIntervalSecond)
                    .GroupBy(m => m.RuleType);

                var tasks = new List<Task>();

                foreach (var group in dueMonitors)
                {
                    foreach (var monitor in group)
                    {
                        tasks.Add(ProcessMonitorAsync(monitor, values, now, ct));
                    }
                }

                await Task.WhenAll(tasks);
            }
            catch (Exception ex)
            {
                _logger.LogError(ex, "Worker 计算循环异常");
            }

            await Task.Delay(200, ct); // 最小调度粒度
        }
    }

    private async Task ProcessMonitorAsync(
        MonitorConfig monitor, IDictionary<string, double?> values,
        DateTime now, CancellationToken ct)
    {
        try
        {
            var state = await _stateStore.GetAsync(monitor.Id);
            var result = _dispatcher.Calculate(monitor, values, state, now);

            if (result.HasEvent)
            {
                var alarm = new AlarmSnapshot
                {
                    MonitorId = monitor.Id,
                    MonitorKey = monitor.Key,
                    MonitorName = monitor.Name,
                    StatusKey = result.State,
                    Value = result.TriggerValue ?? 0,
                    OccurTime = now,
                    ConfigVersion = monitor.LastModificationTime,
                };

                await _alarmWriter.WriteRealtimeAlarmAsync(alarm);
                await _alarmWriter.WriteHistoryAlarmAsync(alarm.ToAlarmEvent());
            }
            else if (string.IsNullOrEmpty(result.State) && state?.PreviousStatus != null)
            {
                // 报警消除
                await _alarmWriter.ClearRealtimeAlarmAsync(monitor.Id);
            }

            monitor.LastCalculateTime = now;
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "处理监控项 {MonitorId} 异常", monitor.Id);
        }
    }
}
```

## 3. 管道详细设计

### 3.1 Worker 内部管道

复用 TPL Dataflow 模式，在 Worker 内部按规则类型做管道特化：

```
Worker 内部 TPL Dataflow 管线:

TrendDB 批量读取
    │
    ├──► Expression Pipeline (MaxDOP=2, BufferSize=10000)
    │    └── TransformBlock → ActionBlock(结果写入)
    │
    ├──► RangeDuration Pipeline (MaxDOP=4, BufferSize=50000)
    │    └── TransformBlock → ActionBlock(结果写入)
    │
    ├──► RangeFrequency Pipeline (MaxDOP=4, BufferSize=30000)
    │    └── TransformBlock → ActionBlock(结果写入)
    │
    ├──► PackageValue Pipeline (MaxDOP=4, BufferSize=20000)
    │    └── TransformBlock → ActionBlock(结果写入)
    │
    ├──► MultiState Pipeline (MaxDOP=6, BufferSize=30000)
    │    └── TransformBlock → ActionBlock(结果写入)
    │
    └──► IO Pipeline (MaxDOP=8, BufferSize=5000)
         └── TransformBlock → ActionBlock(结果写入)
             (WallTemperature + InterfaceMonitoring)
```

### 3.2 管道并行度配置

| 管道 | MaxDOP | BufferSize | 说明 |
|------|--------|------------|------|
| Expression | 2 | 10000 | 纯数学运算，CPU 密集但轻量 |
| RangeDuration | 4 | 50000 | 需查极值缓存，中等复杂度 |
| RangeFrequency | 4 | 30000 | 需遍历频率窗口 |
| PackageValue/FeatureValue | 4 | 20000 | 位解析 |
| MultiStateRangeDuration | 6 | 30000 | 多状态并发，重量级 |
| WallTemperature/IO | 8 | 5000 | IO 密集（外部调用），高并发低缓冲 |

## 4. gRPC 接口设计

### 4.1 Proto 定义

```protobuf
syntax = "proto3";

package ruleengine;

service WorkerService {
    // Master → Worker: 分配监控项
    rpc AssignMonitors(AssignRequest) returns (AssignResponse);

    // Master → Worker: 回收监控项
    rpc RevokeMonitors(RevokeRequest) returns (RevokeResponse);

    // Master → Worker: 健康检查
    rpc HealthCheck(HealthCheckRequest) returns (HealthCheckResponse);
}

service MasterService {
    // Worker → Master: 注册
    rpc Register(RegisterRequest) returns (RegisterResponse);

    // Worker → Master: 心跳
    rpc Heartbeat(HeartbeatRequest) returns (HeartbeatResponse);
}

message AssignRequest {
    repeated MonitorConfigMsg monitors = 1;
    int64 shard_epoch = 2;
}

message MonitorConfigMsg {
    string id = 1;
    string key = 2;
    string name = 3;
    int32 rule_type = 4;
    int32 refresh_interval_second = 5;
    string tag_name = 6;
    bytes rule_options_json = 7;   // JSON 序列化的规则选项
    int64 last_modification_time = 8;
}
```

### 4.2 消息流

```
Worker 启动:
  Worker ──Register──► Master
  Worker ◄──AssignMonitors── Master (初始分配)

运行时:
  Worker ──Heartbeat──► Master (每 10s)
  Master ──AssignMonitors──► Worker (配置变更)
  Master ──RevokeMonitors──► Worker (配置变更)
  Master ──HealthCheck──► Worker (每 30s)

Worker 宕机:
  Master 检测心跳超时 → 故障转移流程 → 其他 Worker 收到 AssignMonitors
```

| 场景 | 策略 |
|------|------|
| TrendDB 读取失败 | 重试 3 次，间隔 1s/2s/4s，全失败则跳过本周期，记录 ERROR 日志 |
| Redis 写入失败 | 重试 3 次，间隔 500ms，全失败则记录 CRITICAL 日志 |
| ClickHouse 写入失败 | 写入本地 WAL 缓冲文件，后台定时重试 |
| RocksDB 读写失败 | 记录 CRITICAL 日志，使用空状态继续计算 |
| gRPC 连接断开 | 指数退避重连 (1s → 2s → 4s → ... → 60s 上限) |
| 配置版本过期 | Worker 拒绝写入，等待 Master 推送新配置 |

### 6.2 日志级别

| 级别 | 使用场景 |
|------|---------|
| Trace | 每个监控项的计算细节 |
| Debug | 管道队列深度、批量操作计数 |
| Information | Worker 注册/注销、配置变更、分片迁移 |
| Warning | 数据源读取失败重试、报警写入重试 |
| Error | 管道异常、gRPC 断连、状态恢复失败 |
| Critical | RocksDB 损坏、TrendDB 连接池全部断开 |

## 7. 安全设计

### 7.1 通信安全

- gRPC: 使用 TLS 1.3（生产环境），开发环境可使用 insecure
- HTTP: JWT Bearer Token 认证
- Redis: 密码认证 + TLS（生产环境）
- ClickHouse: 用户名/密码认证 + TLS（生产环境）

### 7.2 配置安全

- 密码、连接字符串等敏感信息使用环境变量注入，不写入配置文件
- gRPC 通信使用 mTLS（可选，生产环境启用）
