## 环境信息

- Rule Engine Master: http://localhost:11081
- SIS Service: http://localhost:11080
- **正确** Monitor Center API URL: `GET /api/services/monitorcenter/monitorDataForPublic/GetAllMonitors`
- **注意**: ABP 插件模式使用模块前缀 `monitorcenter` 而不是 `app`
- Rule Engine Health: `GET /api/ruleengine/health`
- Rule Engine Sync: `POST /api/ruleengine/monitors/on-changed`

## 测试中发现的问题与修复

1. **MonitorCenterClient DI 双重注册**: `AddHttpClient<T>` 已注册为 typed client，额外的 `AddSingleton<T>` 导致 BaseAddress 为 null。修复：移除 AddSingleton。
2. **ABP 插件 API URL 前缀**: 插件模块的 API 使用模块名 (如 `monitorcenter`) 作为 URL 前缀，而非主应用的 `app`。URL 正则从 DLL 名称提取。
3. **API 方法类型**: ABP 动态 Web API 对无参数的 `Get*` 方法暴露为 HTTP GET，而非 POST。
4. **GetHisEvent 未实现分布式代理**: Monitor Center 中只有 `GetRealEvent` 实现了到规则引擎的代理，历史查询 `GetHisEvent` 和闭环验证暂未代理。
