# 前置规则 (Prerule) 设计文档

> 严格对标 MonitorCenter `PrerulePipeline` + `PreruleCheckBlock` 实现，2026-07-18

## 1. 概念澄清

MonitorCenter 的"前置规则"**不是一个简单的抑制开关**，而是一个**独立的规则计算单元**：

- 前置规则有**自己的数据源** (`MonitorSources`)
- 前置规则有**自己的规则条件** (Expression / RangeDuration / RangeFrequency / PackageValue / MultiStateRangeDuration)
- 前置规则通过独立的 `PrerulePipeline` 计算，结果缓存在 `PreruleStore`
- 监视项通过 `PreruleId` 引用前置规则，**多个监视项可共享同一个前置规则**
- 前置规则计算结果为 `true`（满足条件）或 `false`（不满足条件）

```
┌─────────────────────────────────────────────────────────────────┐
│                      MonitorCenter 架构                          │
│                                                                  │
│  PrerulePipeline (独立管道)                                       │
│  ┌──────────────┐    ┌──────────────────┐                        │
│  │ RuleCalculate │ →  │ PreruleCacheBlock │                       │
│  │ Block         │    │ (缓存 true/false) │                       │
│  └──────────────┘    └────────┬─────────┘                        │
│                               │                                  │
│                    PreruleStore._preruleStateShards               │
│                    { preruleId: bool }                            │
│                               │                                  │
│  MonitorItemPipeline          │                                  │
│  ┌────────────────┐           │                                  │
│  │ PreruleCheck   │ ← 读取 ───┘                                  │
│  │ Block          │   true  → 继续计算                            │
│  │                │   false → "不满足前置条件"                     │
│  └────────────────┘                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 数据模型

### 2.1 前置规则定义 (对标 `PreruleCacheItem`)

```csharp
// 从 MonitorCenter 同步，对应 ssmcprerulemst 表
public class PreruleDefinition
{
    public string Id { get; set; }                    // 前置规则主键
    public string Name { get; set; }                  // 前置规则名称
    public string Desc { get; set; }                  // 描述
    public int RefreshIntervalSecond { get; set; }    // 刷新间隔(s)
    public string FocusSourceId { get; set; }         // 关注数据源 ID
    public bool IsEnabled { get; set; }               // 启用状态
    public RuleType RuleType { get; set; }            // 规则类型

    // 数据源 (独立于监视项)
    public List<MonitorSourceDefinition> MonitorSources { get; set; } = new();

    // 规则条件 — 根据 RuleType 使用对应字段
    public RuleExpressionDefinition? RuleExpression { get; set; }
    public List<RuleRangeDurationDefinition> RuleRangeDurations { get; set; } = new();
    public List<RuleRangeFrequencyDefinition> RuleRangeFrequencies { get; set; } = new();
}
```

    public int Priority { get; set; }
    public bool BreakOnHit { get; set; }

## 3. 核心流程

### 3.1 前置规则评估管道

对标 MonitorCenter `PrerulePipeline`。每个前置规则作为独立的计算单元：

```
Timer (每 RefreshIntervalSecond 触发)
    │
    ▼
┌─────────────────────────────────────────────────┐
│ Step 1: 获取前置规则数据源                          │
│                                                    │
│   PreruleDefinition.MonitorSources                 │
│       ↓                                            │
│   ITrendDataReader 读取实时数据                      │
│       ↓                                            │
│   Dictionary<string, double?> sourceData           │
│   (key = data source Alias, value = current value) │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│ Step 2: 规则计算 (对标 RuleCalculateBlock)          │
│                                                    │
│   switch (RuleType)                                │
│   {                                                │
│     case Expression:                               │
│         → 解析 Code 表达式，代入 sourceData          │
│         → result.State = 表达式为 true 时非空       │
│                                                    │
│     case RangeDuration:                           │
│         → 对每个 RuleRangeDuration:                 │
│           左值 = sourceData[LeftSourceKey]          │
│           右值 = sourceData[RightSourceKey]         │
│           比较 (SymbolType) → 满足则累计时长          │
│           时长 >= DurationSecond → hit              │
│         → result.State = 命中的 MonitorStatusKey    │
│                                                    │
│     case RangeFrequency:                          │
│     case PackageValue:                             │
│     case MultiStateRangeDuration:                  │
│         → 复用现有 RuleDispatcher.Calculate 逻辑    │
│   }                                                │
└────────────────────┬────────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────┐
│ Step 3: 缓存结果 (对标 PreruleCacheBlock)          │
│                                                    │
│   bool state = !result.State.IsNullOrWhiteSpace(); │
│   if (!prerule.IsEnabled) state = true;            │
│                                                    │
│   _preruleStore.SetState(preruleId, state);        │
│   // 写入分片 ConcurrentDictionary                  │
└─────────────────────────────────────────────────┘
```

### 3.2 MonitorItem 前置检查 (对标 `PreruleCheckBlock`)

在 Worker 计算主循环 `ProcessMonitorAsync` 中，**在数据源读取之前**执行：

```
ProcessMonitorAsync(monitor)
    │
    ├── if (!string.IsNullOrEmpty(monitor.PreruleId))
    │   {
    │       bool? state = _preruleStore.GetState(monitor.PreruleId);
    │
    │       if (state == null)
    │           → return;  // 前置规则还没计算完，跳过本轮
    │
    │       if (state == false)
    │       {
    │           // 不满足前置条件：清除报警 + 写历史事件
    │           await WriteClearAlarm(monitor, "不满足前置条件");
    │           return;  // 不执行规则计算
    │       }
    │       // state == true → 继续正常计算
    │   }
    │
    └── 继续现有计算流程...
```

### 3.3 状态恢复

Worker 重启时，前置规则状态需要重新计算。由于 `InMemoryStateStore` 是易失的：

```
Worker 启动
    │
    ├── 从 Master 接收 PreruleDefinitions (通过 ConfigPush)
    │
    ├── 初始化 PreruleStore (空)
    │
    └── 第一轮计算时，所有 Prerule 状态为 null
        → MonitorItem 跳过计算（等待前置规则准备好）
        → 第一个 RefreshIntervalSecond 之后，Prerule 计算结果就绪
```

## 4. 存储设计

### 4.1 PreruleStateStore

对标 MonitorCenter `PreruleStore` 的分片 `ConcurrentDictionary`：

```csharp
public class PreruleStateStore
{
    private const int SHARD_COUNT = 64;
    private readonly ConcurrentDictionary<string, bool>[] _shards;

    public PreruleStateStore()
    {
        _shards = new ConcurrentDictionary<string, bool>[SHARD_COUNT];
        for (int i = 0; i < SHARD_COUNT; i++)
            _shards[i] = new ConcurrentDictionary<string, bool>();
    }

    private ConcurrentDictionary<string, bool> GetShard(string key)
    {
        uint hash = unchecked((uint)key.GetHashCode());
        return _shards[hash & (SHARD_COUNT - 1)];
    }

    public void SetState(string preruleId, bool state)
        => GetShard(preruleId)[preruleId] = state;

    public bool? GetState(string preruleId)
        => GetShard(preruleId).TryGetValue(preruleId, out var s) ? s : null;

    public IReadOnlyDictionary<string, bool> GetAllStates()
    {
        var result = new Dictionary<string, bool>();
        foreach (var shard in _shards)
            foreach (var kvp in shard)
                result[kvp.Key] = kvp.Value;
        return result;
    }
}
```

### 4.2 PreruleDefinitionStore

前置规则定义缓存，对标 MonitorCenter `_preruleCacheItems`：

```csharp
public class PreruleDefinitionStore
{
    private readonly ConcurrentDictionary<string, PreruleDefinition> _definitions = new();

    public void LoadAll(List<PreruleDefinition> definitions)
    {
        _definitions.Clear();
        foreach (var d in definitions)
            _definitions[d.Id] = d;
    }

    public PreruleDefinition? Get(string preruleId)
        => _definitions.TryGetValue(preruleId, out var d) ? d : null;

    public List<PreruleDefinition> GetAll()
        => _definitions.Values.ToList();
}
```

## 6. 配置同步

message ConfigPush {

### 6.2 从 MonitorCenter 拉取前置规则

Master 启动时从 MonitorCenter 拉取两类数据：
1. `GET /api/.../GetAllMonitors` → 监视项列表（含 `preruleId` 字段）
2. `GET /api/.../GetAllPrerules` → 前置规则定义列表（含数据源 + 规则条件）

## 7. 与现有 PrerulePipeline 的关系

现有的 `Worker/Calculation/PrerulePipeline.cs` 中的 `ManualFlag`/`StopMonitor`/`SourceDependency` 三个检查：

- 这是 MonitorCenter `MonitorCacheItem` 上的**三个独立字段**（`ManualFlag`, `StopMonitorKey`, `MonitorSources`）
- 它们在 MonitorCenter 中**不属于前置规则系统**——它们属于 `InterfaceMonitoring` 规则类型的特殊逻辑
- 应保留这三个检查，但**放在前置规则检查之后**，作为 "InterfaceMonitoring 规则类型的附加逻辑"
- 现有的 `PreruleConfig` 类应重命名为 `InterfaceMonitoringConfig` 或类似名称以消除歧义
